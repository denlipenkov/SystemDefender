# GPU и рендер (PixiJS v8)

## Как устроено сейчас

- Кадр боя и меню по-прежнему **рисуются** в память через **Canvas 2D** (`ctx` на offscreen-элементе `canvas`).
- На экран кадр выводится через **PixiJS** с предпочтением **WebGPU** (`preference: 'webgpu'`). Если WebGPU недоступен (драйвер, ОС), Pixi выбирает **WebGL2**.
- В консоли при старте логируется строка `[render] Pixi backend: …` — по ней можно проверить фактический бэкенд.

## Требования

- **Electron**: актуальная ветка Chromium (в проекте — `electron` ^41) с поддержкой WebGPU на целевой платформе.
- **Windows**: обновлённые драйверы GPU; при отсутствии WebGPU игра должна оставаться играбельной за счёт WebGL2.

## Сборка

- В `package.json` в `build.files` включён `node_modules/pixi.js/**/*` — перед `electron-builder` выполните `npm install` в корне репозитория.

## Дальнейшая миграция

- Перенос отдельных сущностей (пули, враги, частицы, «Сеть») в нативные **Pixi `Sprite` / `Graphics` / `ParticleContainer`** снизит нагрузку на CPU и уменьшит объём данных, копируемых в текстуру каждый кадр.
